## 문항 1. 본인의 회사 선택 기준은 무엇이며, HD현대가 어떤 측면에서 그 기준에 적합하다고 생각하는지 기술하시오. (600자)

회사 선택 기준은 두 가지입니다. 첫째, AI가 현장에 실제 배포되는 환경인가. 둘째, AI 연구에 활용할 도메인 데이터가 충분한가입니다.

첫째, HD현대는 AI를 현장에 직접 적용하고 있습니다. 조선 용어 13,000개를 학습시킨 'HD Agent'를 배포했고, AI 관제 HiCAMS를 전 야드에 적용했습니다. CES 2026에서 NVIDIA·지멘스 협력 디지털트윈이 소개되었고, FOS 구축을 추진 중입니다. AIX추진실은 AI를 '사람과 사람, 사람과 기계를 연결하는 매개체'로 정의합니다. 저는 KAMCO에서 자연어 요청과 18개 업무 도구를 연결하는 ReAct Agent를 설계·배포했고, 이 방향과 일치합니다.

둘째, 2억 건 이상의 조선·해양 DB와 수십 년간 축적된 숙련공의 암묵지가 있습니다. 네이버 하이퍼클로바X와 협력하여 이 데이터를 LLM에 적용하고 있고, AIX추진실이 암묵지를 'AI Agent로 전이'하는 명장 에이전트 프로젝트를 추진 중입니다. 저는 KAMCO에서 1,000만 건 규모 RAG 시스템을, 해양 선박부품 프로젝트에서 도메인 데이터 벡터 DB 구축을 경험했기에 이 과제에 기여할 수 있다고 판단했습니다.

## 문항 2 (공통) 본인의 주요 연구경력에 대해 기술하시오. (최대 1,000자 입력가능)

1. 한국자산관리공사(KAMCO) AI 솔루션 개발 — 애자일소다 선임연구원 (2024.10~현재)

Agentic AI 시스템을 전담 설계·구현했습니다. LangGraph 기반 복잡한 분기 구조를 OpenAI Function Calling 기반 ReAct Agent로 전면 재설계하여, 18개 독립 도구(일정, 회의실, 전자결재, 번역 등)의 End-to-End 파이프라인을 구현하고 프로덕션에 배포했습니다. 363개 자동화 테스트에서 평균 응답시간 3.9초를 달성했습니다.

약 1,000만 건 규모 RAG 시스템 구축에 참여하여 답변 정확도 92%, Hitrate@5 91%를 달성했습니다. 문서 요약 및 회신문 생성 기능을 단독 개발하여 G-EVAL 4.3/5.0의 품질 평가를 받았습니다.

2. 도메인 특화 LLM 구축 — KAMCO 프로젝트 (2024.10~현재)

Gemma 27B의 금융 도메인 Fine-Tuning을 전담했습니다. 20만 건 KMS 청크를 Contextual Retrieval 방식으로 전처리하고, 질문+검색청크+답변 형식의 학습 데이터를 구축했습니다. RAG 답변에 <reasoning> 태그를 학습시켜 사고 과정을 출력하도록 유도하고, G-EVAL 평가 체계를 설계했습니다.

3. 해양 선박부품 AI 챗봇 — 개인 프로젝트 (2026.01~현재)

해양 부품 도메인에 RAG(pgvector + OpenAI Embedding)과 Gemini 2.5 Flash 기반 다국어(한/영/중) AI 챗봇을 개발했습니다. FastAPI 백엔드 5개 모듈로 설계하고 Railway CI/CD로 배포했습니다. 관리자 대시보드를 HTML/CSS/JS로 직접 구현했습니다.

4. 석사 논문 — NLP 기반 불균형 텍스트 분류 (2024, 부산대학교 통계학과)

SMOTE/B-SMOTE/ADASYN 오버샘플링이 텍스트 분류에 미치는 영향을 분석하여 클래스 비율 최적화 방법론을 제시했습니다.

## 문항 3. 연구 직무에 요구되는 핵심역량은 무엇이라고 생각하며, 그것을 갖추기 위해 어떠한 노력을 기울였는지 기술하시오. (600자)

핵심역량은 두 가지라고 생각합니다. LLM 기반 Agent 설계 역량과 도메인 데이터를 LLM에 연결하는 역량입니다.

첫째, KAMCO 프로젝트에서 Agentic AI 아키텍처 의사결정을 주도했습니다. 초기 LangGraph 기반 파이프라인은 기능 추가 시 라우팅 로직이 복잡해져 유지보수가 어려웠고, "A 직원을 찾아서 해당 직원에게 메일 초안을 작성해줘"같은 다중 요청을 처리하지 못했습니다. 이 한계를 경험한 뒤 ReAct 패턴을 연구하여, Function Calling 지원 모델 전환과 ReAct Agent 재설계를 직접 제안하여 프로덕션에 적용했습니다.

둘째, 도메인 데이터를 LLM에 연결하는 방법을 RAG와 Fine-Tuning 두 방향으로 경험했습니다. KAMCO RAG 구축 과정에서 벡터 검색의 한계를 파악했고, 이를 보완하기 위해 Gemma 27B Fine-Tuning으로 확장했습니다. 학습 데이터 구축 시 단순 청크로는 효과가 낮아 Contextual Retrieval로 문맥을 추가하고, <reasoning> 태그로 사고 과정을 유도하는 방식을 설계했습니다. 해양 프로젝트에서는 pgvector로 RAG를 설계했습니다.

## 문항 4. 살아오면서 가장 도전적이었던 경험과 이를 통해 무엇을 성취할 수 있었는지 기술하시오. (600자)

프론트엔드 경험 없이 해양 선박부품 AI 통합 웹서비스를 혼자 구축한 것이 가장 도전적인 경험이었습니다.

아버지의 선박부품 사업에 AI 상담 서비스를 만들어보자는 생각으로 시작했습니다. 프론트엔드와 클라우드 배포 경험이 없었고 혼자 진행해야 했습니다. 난관은 Railway에 FastAPI 서버를 띄우고 PostgreSQL DB를 연결하는 인프라 구성과, 프레임워크 없이 HTML/CSS/JS만으로 관리자 대시보드(상품 관리, RAG 문서 관리, 메일, 통계 등)를 직접 구현하는 것이었습니다.

동료가 없는 환경에서 AI를 활용하여 해결했습니다. DB 스키마, API 구조, 프론트-백엔드 연동, 배포 설정 등 의사결정이 필요할 때마다 AI와 대화하며 방향을 잡고 검증했습니다. 결과적으로 FastAPI 백엔드 5개 모듈(상품 관리, AI 메일, RAG 채팅, 고객 문의, 공개 API), pgvector 기반 다국어 RAG 챗봇, CI/CD 자동 배포까지 갖춘 서비스를 완성했습니다.

경험 없는 기술 영역도 AI를 활용하면 독립적으로 해결할 수 있다는 것을 확인했고, HD현대에서 새로운 AI 과제에 빠르게 적응하는 데 활용할 수 있습니다.
