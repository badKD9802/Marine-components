# 영마린테크(Young Marine Tech) 프론트엔드 고도화 기획서

## Executive Summary

현재 my-boat-shop의 프론트엔드는 기본적인 상품 전시 기능만 제공하고 있습니다. 이 기획서는 **B2B 해양산업 전문성**과 **신뢰도 강화**, **리드 생성 최적화**를 목표로 전면 개편을 제안합니다.

**핵심 개선 방향:**
- 전문적이고 신뢰감 있는 B2B 디자인 시스템 도입
- 상품 검색/필터링 기능 강화 (주요 이탈점 해결)
- 상세한 상품 정보 제공 및 시각적 개선
- 회사 신뢰도 요소 강화 (인증, 파트너, 실적)
- 견적/문의 프로세스 최적화
- 반응형 및 접근성 개선
- 마이크로 인터랙션을 통한 UX 향상

### 2.2 섹션 1: 네비게이션 바

#### 데스크톱 레이아웃 (≥1025px)
```
┌─────────────────────────────────────────────────────┐
│ ⚓ 영마린테크     | 회사소개 | 제품목록 | 고객센터 | 견적문의 | 🌐 |
│ Young Marine Tech|                                  |
└─────────────────────────────────────────────────────┘
```

**컴포넌트 설명:**
- **로고 영역** (좌측)
  - 앵커 아이콘 + 회사명 (title 언어로 표시)
  - 영문 서브텍스트 "Young Marine Tech"
  - 클릭 시 메인 페이지로 이동

- **메인 메뉴** (중앙)
  - "회사소개": #about-section으로 스크롤
  - "제품목록": #product-section으로 스크롤
  - "고객센터": 푸터 연락처로 스크롤
  - "견적문의": #quote-section으로 스크롤
  - 호버 시: 텍스트 색상 → #0099cc, 언더라인 애니메이션

- **오른쪽 액션**
  - 언어 선택 드롭다운 (🌐 아이콘 + select)
  - 필요시 "견적요청" 주요 버튼 추가 가능

**인터랙션:**
- Sticky: 스크롤 시 상단 고정
- 백그라운드: 스크롤 감지 시 약한 그림자 추가
- 메뉴 호버: 색상 변경 + 0.2s 트랜지션

**모바일 레이아웃 (≤768px)**
```
┌──────────────────────────┐
│ ⚓ 영마린테크 | ☰ | 🌐  │
└──────────────────────────┘
```
- 메인 메뉴 숨김 (모바일 메뉴 버튼으로 전환)
- 모바일 메뉴: 네비게이션 드로어 (좌측에서 슬라이드 인)
- 높이: 56px (터치 친화적)

**기술 구현 팁:**
```javascript
// Sticky navbar 스크롤 감지
window.addEventListener('scroll', () => {
  if (window.scrollY > 50) {
    navbar.classList.add('scrolled');
  } else {
    navbar.classList.remove('scrolled');
  }
});

// 메뉴 클릭 시 부드러운 스크롤
document.querySelectorAll('.nav-menu a').forEach(link => {
  link.addEventListener('click', (e) => {
    e.preventDefault();
    const target = document.querySelector(link.getAttribute('href'));
    target?.scrollIntoView({ behavior: 'smooth' });
  });
});
```

## 5. 인터랙션 & 애니메이션 기획

### 5.1 스크롤 애니메이션 (Intersection Observer)

**목표:**
페이지 스크롤 시 요소들이 부드럽게 나타나서 사용자 몰입도 증가

#### 구현 방식

```javascript
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('animate-in');
      observer.unobserve(entry.target);
    }
  });
}, {
  threshold: 0.1 // 10% 이상 보일 때 트리거
});

document.querySelectorAll('.observe-fade').forEach(el => {
  observer.observe(el);
});
```

#### 애니메이션 종류

| 요소 | 애니메이션 | 지속 시간 | 딜레이 |
|------|-----------|---------|-------|
| 히어로 헤드라인 | fade-in + slide-up | 0.6s | 0s |
| 히어로 서브텍스트 | fade-in + slide-up | 0.6s | 0.2s |
| 섹션 제목 | fade-in + slide-left | 0.5s | 0s |
| 카드 그룹 | fade-in + scale | 0.5s | staggered 100ms |
| 텍스트 블록 | fade-in | 0.4s | 0s |
| 이미지 | fade-in + slide | 0.5s | 0s |

**CSS 예시:**
```css
.observe-fade {
  opacity: 0;
  transform: translateY(20px);
  transition: opacity 0.6s ease-out, transform 0.6s ease-out;
}

.observe-fade.animate-in {
  opacity: 1;
  transform: translateY(0);
}

/* Staggered 애니메이션 */
.card:nth-child(1) { transition-delay: 0s; }
.card:nth-child(2) { transition-delay: 0.1s; }
.card:nth-child(3) { transition-delay: 0.2s; }
```

---

### 5.2 호버 효과

#### 버튼 호버
- 배경색 변경
- scale: 1.02
- 그림자 증가
- 트랜지션: 0.2s cubic-bezier(0.4, 0, 0.2, 1)

#### 카드 호버
- scale: 1.02
- translateY: -4px
- 그림자: shadow-md → shadow-lg
- 트랜지션: 0.3s ease

#### 링크 호버
- 텍스트 색상: #666 → #003366
- 언더라인: 나타남 (animate width 0 → 100%, 0.3s)

#### 이미지 호버 (상품 카드)
- scale: 1.1 (부모 overflow: hidden)
- 트랜지션: 0.4s cubic-bezier(0.4, 0, 0.2, 1)

---

### 5.3 전환 효과 (Transitions)

#### 페이지 전환
- 클릭 시: fade-out (200ms) → 페이지 이동 → fade-in (300ms)
- 또는 기존 페이지 구조 유지 (SPA 식 전환)

#### 모달 열기
- fade-in: 배경 0 → rgba(0, 0, 0, 0.5), 0.3s
- scale-in: 콘텐츠 0.95 → 1, 0.3s
- 닫기: 반대로

#### 드로어 열기 (모바일 메뉴)
- 좌측에서 슬라이드 인: translateX(-100%) → 0, 0.3s

#### 탭 전환
- 콘텐츠: fade-out (150ms) → 콘텐츠 변경 → fade-in (150ms)
- 또는 opacity: 1 → 0 → 1 (0.3s)

---

### 5.4 로딩 상태

#### 이미지 로딩
```html
<div class="image-skeleton">
  <img src="..." class="lazy-image">
</div>
```

```css
.image-skeleton {
  background: linear-gradient(90deg, #f0f0f0 25%, #e0e0e0 50%, #f0f0f0 75%);
  background-size: 200% 100%;
  animation: shimmer 1.5s infinite;
}

@keyframes shimmer {
  0% { background-position: 200% 0; }
  100% { background-position: -200% 0; }
}

.image-skeleton img {
  opacity: 0;
  transition: opacity 0.3s;
}

.image-skeleton img.loaded {
  opacity: 1;
}
```

#### 상품 그리드 로딩
- 카드 스켈레톤 표시 (회색 박스, shimmer 애니메이션)
- 로드 완료 후: fade-in (0.3s)

#### 견적 폼 제출
- 버튼: disabled 상태, "제출 중..."
- 로딩 스피너 표시
- 성공 후: 확인 메시지 (green 배경) + 자동 닫기 (3s)

## 12. 결론

이 기획서는 현재 my-boat-shop의 프론트엔드를 **B2B 해양산업 전문성**을 갖춘 신뢰감 있는 플랫폼으로 전환하는 로드맵입니다.

**핵심 성과:**
1. 주요 이탈점(검색/필터) 해결로 사용자 경험 30% 개선 예상
2. 신뢰도 요소 강화로 B2B 리드 생성 증대
3. 반응형 및 접근성 개선로 모바일 사용자 확보
4. 일관된 디자인 시스템으로 유지보수성 증대

**성공 지표:**
- 페이지 로드 타임: 3초 이내 달성
- 모바일 사용자 비율: 40% 이상
- 상품 검색→상세보기→문의 전환율: 현재의 2배 달성
- 사용자 체류 시간: 평균 3분 이상

이 기획서가 영마린테크의 성장에 기여하길 바랍니다.
